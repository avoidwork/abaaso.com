 What's a DataStore?

DataStores are a NoSQL database in RAM, which can provide RESTful abstraction of a Server API (collection).

1-n relationships exists with DataLists and DataGrids, e.g. each view can be filtered and/or templated differently.

All I/O methods return Deferreds.

 View of a DataStore?

Reactive views can be created with DataLists or DataGrids. DataGrids provide the "classic" sortable column/row output, while DataLists create templated UL lists.

The DOM is created with interoperability in mind.

Filtering via input field, as well as case sensitive, case insensitive, & mixed sensitivity sorting are optional features.

 How do I do AJAX?

Strings have the HTTP verbs as methods, e.g. '"http://abaaso.com".get(success, failure);', providing a natural syntax for common Server interactions.

Deferreds are returned from these methods.

 How do I use MongoDB?

DataStores can be wired to MongoDB by setting a connection string for "data.mongodb".

You should enable "data.autosave" so MongoDB is updated when the DataStore changes. The ID ("data.parentNode.id") will be the name of the Collection in MongoDB, and the "key" will be "_id".

Data can be retrieved with "data.restore()", and saved with "data.save()".

 Why not MVC?

JavaScript is a prototypal OOP language, and I think it makes the most sense to follow that design.

Plus, there's a ton of great MVC frameworks out there already.

 Deferreds / Promises?

Promises are a design pattern for handling async I/O, providing an eventual outcome. Promises return new Promises, creating a hierarchical relationship that can be forked by returning a new Promise, and reconciled from either Parent or Child.

Deferreds are extended Promises with more behaviour, allowing tail calls upon "done" or "fail", & "always".

 Why extend prototypes?

abaaso's nature is to be everywhere. It hooks into the native Objects to fill perceived gaps of functionality, and offer it's own subset.

It's done in a "safe" way so any other library or framework can "take" the hook for itself.

 How is this different?

abaaso provides most of the features of retail application frameworks at a fraction of the payload size, while being completely free.

It isn't meant to replace jQuery or zepto, it's an alternative to Sencha ExtJS or KendoUI. It's self reliant and DOM centric, allowing you to easily leverage UI frameworks like Bootstrap.

You can get most of the feature set by combining popular libraries together, but the result won't work as smoothly.
